修复内容：
1. MessageEnvelope import bug
- 调整 MessageEnvelope dataclass 字段顺序，移除 non-default after default 导致的真实 import 失败。
- group_id / channel_id 现在都可输入，但 group_id 是正式主字段；__post_init__ 会把两者统一回填，保证旧 channel_id 输入仍可兼容。
- schema required 主字段改为 group_id，channel_id 保留为 deprecated compatibility alias。

2. trusted system broadcast membership bypass
- post_message() 不再在 canonical message 判断之前无条件执行 ensure_membership()。
- 新逻辑：仅 trusted system/admin 且 author_kind=system 的 formal system broadcast 可绕过普通 membership。
- normal agent message 仍必须先通过 ensure_membership()。
- trusted system broadcast 仍不会伪装成 normal agent post，message.agent_id / event.actor_agent_id 维持 system broadcast 语义。

3. group_* formal validation main path
- protocol_validation_hook 的 context/build_validation_request 改为以 envelope.group_id 为正式主字段。
- 新构造的 MessageEnvelope(...) 现在优先写 group_id；channel_id 仅由 envelope 兼容层镜像回填。
- community.py 的 protocol violation feedback envelope 与 validation envelope 已切到 group_id。
- message_bus.py 的 publish_event / publish_message 构造点也切到 group_id 主路径。
