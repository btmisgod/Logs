本轮只处理 server 侧 plain event publish 收口，不触碰 skill/webhook/frontend，也未做 live workflow 验证。

本轮完成：
1. event publish 链已改为 plain snapshot 主线
- 在 myproject/app/services/event_bus.py 增加：
  - plain_event_snapshot(event)
  - stabilize_event_snapshot(session, event)
- publish_event(...) 现在在入口先转成 plain event snapshot，再把 plain 数据传给：
  - WebProjector
  - Redis publish
  - group webhooks
  - agent webhooks
  - ConsoleProjector
- downstream publish 层不再直接消费 SQLAlchemy ORM Event。

2. WebProjector 改为 plain event envelope 构建
- myproject/app/projectors/web.py::to_publishable() 现在参数是 plain event dict。
- 不再做 EventEnvelope.model_validate(orm_event)。
- 改为直接用 plain event data 构造 EventEnvelope。

3. ConsoleProjector 改为 plain event 消费
- myproject/app/projectors/console.py::project() 现在参数是 plain event dict。
- 不再依赖 ORM Event 属性访问。

4. community.py 直接 publish_event 调用已全部收口
- 在 myproject/app/services/community.py 新增 _publish_plain_events(session, *events)。
- 该 helper 先做 stabilize_event_snapshot(session, event)，再调用 publish_event(plain_snapshot)。
- community.py 内所有直接 publish_event(...) 出口已改成经由该 helper，未留下 mixed ORM/plain publish path。
- refresh 在这里仅作为 snapshot stabilization 使用，不再让 publish 层吃 ORM Event。

修改文件：
- myproject/app/services/event_bus.py
