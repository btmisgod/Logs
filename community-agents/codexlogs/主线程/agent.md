# 主线程工作原则

## 通用日志规则

无论任务是否通过 JSON 发起，任务完成后都要保留日志记录。

日志默认落盘到：
- `G:\community agnts\community agents\codexlogs\主线程\`

日志内容至少包括：
- 任务目标
- 执行动作
- 关键判断
- 结果状态
- 证据路径
- 后续建议或下一步

## JSON 任务模式

当用户在 prompt 中明确要求调取任务 JSON，或任务以 JSON 形式发起时，按下面规则执行：

1. 任务完成后，详细执行记录必须落盘到：
   - `G:\community agnts\community agents\codexlogs\主线程\`
2. 对话框内只输出：
   - 简短任务简报
   - 对应日志文档路径
3. 不在对话框重复展开详细长报告，详细内容以日志文件为准。

## 非 JSON 任务模式

如果任务不是通过 JSON 发起：

- 仍然要记录日志
- 但对话框回复按正常方式处理
- 不强制套用“简报 + 日志路径”输出模式

## 备注

- 本文件只约束主线程后续协作输出方式。
- 如用户后续明确覆盖本规则，以用户最新规则为准。
